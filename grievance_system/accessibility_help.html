Our commitment

SDGRS is designed to be usable by as many people as possible. We aim to align with recognised accessibility standards such as WCAG 2.1 Level AA and continuously improve based on feedback.

Inclusive by design

We consider accessibility from the early design stages of new features, rather than as an after‑thought.

Built‑in accessibility features

  • Logical heading structure and clear labels on all interactive controls.
  • Keyboard‑friendly navigation with visible focus indicators.
  • High‑contrast colour palette suitable for low‑vision users.
  • Forms with meaningful error messages and instructions.

Using the SDGRS portal

You can access SDGRS through modern browsers, screen readers and assistive technologies. If you use a keyboard or switch device, all essential actions can be completed without a mouse.

Tips for easier use

  • Use the “Skip to main content” link at the top of each page to avoid repeated navigation.
  • Adjust text size and zoom in your browser without breaking the layout.

Getting additional help

If you need information in an alternative format, such as large print or a readable PDF, contact our support team.

Reporting accessibility issues

If you encounter an accessibility problem, please tell us which page, what went wrong and the assistive technology you use. This helps us fix issues faster.

Accessibility feedback form will be available in the next release.
